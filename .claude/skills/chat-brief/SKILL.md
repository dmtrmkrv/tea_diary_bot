# Чат-бриф LeafPulse

Задача: полностью переписать `docs/LEAFPULSE_STATUS.md` свежим дайджестом состояния продукта. Файл читает Claude Chat (проект LeafPulseApp) как источник истины при обсуждении продвижения, постов и продуктовых идей — пиши для этого читателя, а не для разработчика.

## Откуда брать данные

1. Память проекта: `MEMORY.md` (индекс) и релевантные `project_*`-файлы — статусы «НА ПРОДЕ», каналы, метрики, планы.
2. `git log --oneline --since=<дата прошлого брифа>` на `origin/main` — что реально доехало до прода.
3. Todoist (проект «📱LeafPulse – App»): секция «Сделано» за период + открытые задачи 1-й очереди — ближайшие планы.
4. Дата прошлого брифа — в шапке текущего файла.

## Структура файла

- **Шапка**: дата обновления, одна строка «что такое LeafPulse».
- **Продукт сейчас** — что умеет приложение (языком пользователя, без истории «как делали»).
- **Новое за период** — 5–10 пунктов крупняка, как для анонса.
- **Цифры и каналы** — аудитория, регистрации, работающие каналы привлечения, конверсии; UTM-схема для ссылок в постах.
- **Ближайшие планы** — что в работе/очереди (без обещаний дат).
- **Не упоминать публично** — актуальный список чувствительного.

## Правила

- Простой язык, без жаргона, коммитов, путей файлов и имён внутренних компонентов.
- НИКОГДА не включать: детали безопасности/уязвимостей (даже починенных — без подробностей механики), топологию инфраструктуры, env-переменные, точные малые цифры аудитории в формулировках «для публикации» (в разделе цифр — можно, с пометкой «не для публики»), причины проблем с Telegram (ТСПУ) в публичных формулировках.
- Бот: сворачивается, в коммуникации не упоминать как живой канал; «перенос записей из бота на сайте» — можно и нужно упоминать.
- Полная перезапись файла (не дописывание), объём — 1–2 экрана.

## После обновления

1. Закоммитить в `staging` (сообщение «docs: чат-бриф от <дата>») и запушить (`git push origin staging`).
2. Напомнить Диме: заменить файл в knowledge проекта LeafPulseApp в Claude Chat (удалить старый → загрузить новый `docs/LEAFPULSE_STATUS.md`).
