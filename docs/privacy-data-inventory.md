# Инвентаризация персональных данных — LeafPulse / TeaNotesBot

> Назначение: основа для составления **Политики обработки персональных данных** (152-ФЗ).
> Собрано **по коду** (модели, auth, Метрика, куки/хранилище) — состояние на 2026-06-26.
> ⚠️ Не юридический документ. Финальные формулировки сверить по шаблону / с юристом.

Сервис состоит из двух частей с **общей БД**: веб-приложение (`leafpulse.ru`) и Telegram-бот (`@TeaNotesBot`). Политика покрывает обе.

---

## A. Идентификация и аккаунт (из Telegram при входе)
| Данные | Хранится? | Примечание |
|---|---|---|
| Telegram ID (числовой) | ✅ да | ключ аккаунта (PK) |
| Имя (`first_name`, ≤64) | ✅ да | |
| Username (`@…`) | ✅ да | нормализуется, ≤32 |
| URL аватара (`photo_url`) | ✅ да | ссылка на CDN Telegram; **сам файл не храним** |
| `last_name` | ❌ нет | приходит, используется только для проверки подписи |
| `auth_date`, `hash` | ❌ нет | только проверка подписи Telegram |

## B. Технические данные и настройки
- **Часовой пояс** (`tz_offset_min`) — автоопределяется из браузера при входе.
- **Кука `token`** — JWT-сессия, срок **180 дней**.
- **localStorage**: `tzoff` (кэш пояса, чтобы не дёргать API), `cookie_consent` (появится с куки-баннером).

## C. Пользовательский контент (создаёт сам пользователь — веб и бот)
- **Дегустации**: название, дата, граммовка, температура, ароматы/вкус/послевкусие (дескрипторы + свободный текст), проливы, оценки, заметки, сценарии/ощущения.
- **Коллекция**: сорта (название, категория, год, регион, поставщик, заметки, остаток) и посуда (название, тип, объём, материал, регион, заметки).
- **Фотографии** (дегустаций и обложки сортов/посуды) — файлы в S3.

## D. Аналитика — Яндекс.Метрика (только прод)
- Конфиг в коде: `clickmap:true, trackLinks:true, accurateTrackBounce:true`.
- **Вебвизор в коде НЕ включён** (подтвердить в настройках счётчика).
- Куки Метрики (`_ym_*`), обезличенные данные о посещениях/переходах/кликах, тех. данные браузера/устройства.
- ⚠️ Утверждение «**IP не сохраняется**» из кода **не подтверждается** — это настройка счётчика Метрики. Проверить в настройках, прежде чем писать в Политике.

## Где хранится
БД PostgreSQL + S3 (фото) — **Timeweb Cloud, Санкт-Петербург (РФ)**.
→ Данные локализованы в РФ — **152-ФЗ ст.18 ч.5 соблюдается** (плюс).

## Третьи лица (получатели/обработчики)
- **Яндекс** (ООО «Яндекс») — Метрика (аналитика).
- **Timeweb** (ООО «Таймвеб») — хостинг/инфраструктура (БД + S3).
- **Telegram** — провайдер идентификации (источник профиля; аватар грузится с серверов Telegram).

## Цели обработки
Аутентификация и ведение аккаунта; хранение и отображение дегустаций/коллекции/фото; сохранение настроек (пояс/сессия); обезличенная аналитика для улучшения сервиса.

## Правовые основания
- Согласие субъекта — ст.6 ч.1 п.1 (чекбокс при входе).
- Исполнение договора / обращение пользователя — ст.6 ч.1 п.5 (предоставление сервиса).
- Аналитика — согласие (куки-уведомление).

## Сроки хранения
Пока существует аккаунт / до отзыва согласия / до удаления пользователем.
Бэкапы БД — ежедневные, 7 копий (выпадают из ротации ~7 дней). Кука `token` — 180 дней; куки Метрики — по срокам Яндекса.

## Права субъекта и отзыв согласия
Доступ, исправление, удаление, отзыв согласия, жалоба в РКН.
Уже реализовано: **экспорт CSV**, редактирование/удаление отдельных записей.

## ⚠️ Пробелы (учесть при составлении документа)
1. **Нет удаления аккаунта целиком** — удаляются только отдельные записи. «Право на удаление / отзыв согласия» пока обслуживается по email-запросу. → заведена задача на удаление аккаунта (p1).
2. **«IP не сохраняется» по Метрике** — проверить в настройках счётчика.
3. **Кука `token` не HttpOnly** (ставится через `document.cookie`) — JWT доступен из JS (риск увода при XSS). Это безопасность, не Политика; кандидат в отдельную задачу.

## Подход к будущему Яндекс-входу
Описывать только **текущее** состояние (Telegram). Добавить нейтральную строку: «способы входа могут дополняться (напр., Яндекс ID, email); Политика будет обновлена». Конкретику (email как новые ПДн, Яндекс как источник) вносить, когда фича выйдет.
